# Shot Template Studio

> AI 動画生成のための再利用可能なショット・テンプレート・ライブラリ。
> Kling、Runway、Pika、Sora、Midjourney Video 等、主要な動画モデルに対応した自然言語テンプレート。

---

## 背景

AI 動画生成には「記憶の問題」がある。

複数人物の複雑なショット——一人称視点で横から別のキャラに割り込まれる瞬間、雨中の無言の別れ、擦れ違いながら振り返る一瞬——を構想するたびに、プロンプトをゼロから書き起こすことになる。一度のプロンプトに注いだ工夫は、使い終われば蒸発する。

**Shot Template Studio は、その工夫を資産に変える。**

ショットは一度書けば、異なる人物・異なる場面・異なるプロジェクトで繰り返し再利用できる。

---

## 仕組み

ショットは、画像プレースホルダーを含むプレーンテキスト・テンプレートとして保存される：

```
【横取り】

必要な画像：
- {場面}：ロケーション
- {キャラB}：遠くから走ってくる人物
- {キャラC}：横から割り込む人物
- {キャラD}：一人称視点の「私」

---

{場面}を背景に。{キャラD}の一人称視点。

{キャラB}が画面の奥から手を振りながらカメラへ走ってくる。
被写界深度は{キャラB}にフォーカス。

{キャラD}がゆっくり左手を上げ、手のひらを上に向ける。

ここで{キャラC}の右手が画面左から現れ、{キャラD}の左手を握る。
続いて{キャラC}が画面左から歩み出て、背後の{キャラB}を遮る。
被写界深度が手前に引かれる。

{キャラC}の上半身が画面の大半を占め、切ない目でカメラを見つめ、
少し首を傾げて微笑する。両手で{キャラD}の手を掬い、
自分の頬に当て、目を細めて余韻に浸る。

---

用途：感情の反転、予期せぬ介入、三角関係
長さ：8〜12秒
```

使用時には 4 枚の参照画像を提供する。skill が各プレースホルダーを画像の説明に置き換え、そのまま使える動画プロンプトを出力する。

---

## 双方向ワークフロー

```
テンプレート → プロンプト：
  /shot use <テンプレート>
    → 参照画像を提供
    → 貼り付け可能なプロンプトを取得

動画 → テンプレート：
  /shot extract <動画または説明>
    → AI がショット構造を分析
    → 再利用可能なテンプレートを生成
    → 個人ライブラリに保存
```

このループこそが中核的価値：**気に入ったショットは蓄積され、書いたプロンプトは共有される。**

## インストール

```bash
git clone https://github.com/zuiho-kai/shot-template-studio.git
cp shot-template-studio/.claude/commands/shot.md ~/.claude/commands/
```

`templates/` 以下のテンプレートは自動的に認識される。

## コマンド

### `/shot list`
全テンプレートをタグと用途付きで一覧表示。

### `/shot use <名前>`
対話的にプロンプトを構成。各プレースホルダーに参照画像を指定すると、主要な動画生成モデルに対応する自然言語プロンプトを出力。

### `/shot extract <ソース>`
動画ファイルまたは説明文を分析し、新規テンプレートを生成。必要画像、ショット配列、ペース、用途タグを AI が自動抽出。

### `/shot save <名前>`
対話中に書いた（あるいは練った）プロンプトを再利用可能なテンプレートとして保存。具体的な人物・場面を自動的にプレースホルダーへ抽象化。

### `/shot ui`
ビジュアル Web インターフェースを起動。ローカルサーバーを立ち上げ、ブラウザを自動で開く。

## Web UI

非技術ユーザー向けのビジュアル・インターフェース。コード不要。

```bash
python server.py
# http://localhost:8090 で開く
```

機能：
- 全テンプレートをビジュアルに閲覧
- テンプレート新規追加（`.txt` 内容を貼り付け、またはファイルをアップロード）→ ディスクに直接書き込み
- テンプレート削除（`.txt` ファイルを削除）
- テンプレート・エクスポート（`.txt` をダウンロードして共有）
- AI アシスト生成（`/shot extract` コマンドをクリップボードにコピー）
- 各プレースホルダーに参照画像番号を割り当て
- ワンクリックでプロンプトをコピー（Kling / Runway / Pika 対応）

テンプレートは `templates/` 内の `.txt` ファイルとして永続化。テンプレートの共有 = `.txt` ファイルを送るだけ。

## 設計思想

**自然言語、構造化データではなく。**

対象ユーザーは (a) 必ずしもコードを書かないクリエイティブ職、(b) 訓練分布が自然言語である動画生成モデル。中間の構造化表現——JSON、YAML、DSL——は前者には認知的負担、後者には翻訳コストとなる。

**プレースホルダーは母語で。**

日本語のユーザーにとって `{キャラA}` は `{character_a}` より直感的。テンプレートは多言語対応が基本。

**エディター・フレンドリー。**

全テンプレートは `.txt` ファイル。Git に不慣れな演出家でもメモ帳で開いて、文言を調整し、保存して使える。パースエラーもスキーマ違反もない。

**モデル非依存。**

コンパイルされたプロンプトは純粋な記述テキストで、Kling、Runway Gen-3、Pika、Sora、Hailuo、Midjourney Video 全てで一貫して動作。モデル別の調整不要。

## マーケットプレイスのビジョン

個人のテンプレートライブラリは線形成長。共有マーケットプレイスは指数的成長。

こんなキュレート・コレクションをブラウズできると想像してほしい：

- **ウォン・カーウァイ** — 長尺クローズアップ、反射面、ネオン配色
- **クリストファー・ノーラン** — IMAX スケールの実写構図、パラレル・エディティング
- **韓国ドラマ集** — 運命の出会い、雨の告白、スローモーション視線
- **ドキュメンタリー集** — インタビュー構図、B-roll 転場、カット・アウェイ

コミュニティ・テンプレートは中央の `shot-template-marketplace` リポジトリへの PR として提出。キュレーターが品質を審査。ユーザーがブラウズ、インストール。テンプレート作者はポートフォリオを築き、演出家はプロジェクトごとに数時間を節約。

## 関連プロジェクト

- **[continuous-scene-designer](https://github.com/zuiho-kai/continuous-scene-designer)** — 姉妹プロジェクト。マルチエージェント画像編集の空間的一貫性を扱う。後者は静止画ワークフロー、本プロジェクトは動画ショット・ワークフロー。

| | continuous-scene-designer | shot-template-studio |
|---|---|---|
| ドメイン | 静止画編集 | 動画ショット構成 |
| 状態表現 | 構造化 JSON | 自然言語テンプレート |
| 主要ユーザー | マルチエージェント・フレームワーク | 演出家・クリエイター |
| 成果物 | `scene_state.json` | `<ショット名>.txt` |

共通原則：**創造的判断を再利用可能な状態として保存する**。
