# 从视频提取模板的示例

演示 `/shot extract` 的完整工作流程。

---

## 场景

你在抖音刷到一段喜欢的镜头：一个女生在咖啡馆看书，
男生推门进来找到她，默默把一束花放在桌上。

你觉得这个镜头很适合自己的短片，想存下来复用。

---

## 正常流程：不用模板库

```
看到视频 → 截图参考 → 下次自己写 prompt 
→ 写了一半发现忘了细节 → 再翻回去看 → 反复对照
→ 花 30 分钟才写出差强人意的 prompt
→ 下次又要重来一次
```

痛点：**每次都重来，经验不积累**。

---

## 用 shot-template-studio

### Step 1: 让 AI 看视频

```
用户: /shot extract
      [附上视频文件或描述]
      这段视频：一个女生在咖啡馆看书，男生推门进来环顾
      四周找到她，走过去把一束花放在桌上。
```

### Step 2: AI 拆解镜头

AI 会自动识别：
- 需要几张图（场景 1 张 + 角色 2 张）
- 镜头序列（开场 → 环顾 → 走近 → 放花 → 反应）
- 节奏和情绪
- 适用场景

### Step 3: 输出模板草稿

```
【咖啡馆送花】

需要的图：
- 场景：咖啡馆内景
- 角色A：看书的女生
- 角色B：送花的男生

---

以{场景}为背景。

{角色A}坐在靠窗的位置，低头看书，手边放着一杯咖啡。
画面安静，只有书页翻动的声音。

镜头切到门口，{角色B}推门进来，带进一阵风，环顾四周。
{角色B}的目光停在{角色A}身上，露出一点紧张的笑意。

{角色B}穿过咖啡馆，走到{角色A}桌前。
{角色A}察觉到有人，抬起头。

{角色B}把一束花轻轻放在桌上，没有说话，只是微笑。
镜头定格在{角色A}的表情上——从惊讶到温柔。

---

适用：浪漫、惊喜、安静的情感瞬间
时长参考：10-14秒
```

### Step 4: 你确认保存

```
用户: 保存为"咖啡馆送花"

AI: [写入 templates/咖啡馆送花.txt]
    模板已保存。以后用 /shot use 咖啡馆送花 即可复用。
```

### Step 5: 下次直接复用

```
几天后，另一个项目：
用户: /shot use 咖啡馆送花
      [上传你项目的场景图 + 两个角色图]

AI: [输出替换好占位符的完整 prompt]

用户把这段 prompt 粘贴到 Kling/Runway
→ 直接得到风格一致的视频
```

## 能积累到什么程度

用了一年后，你的 `templates/` 目录可能长这样：

```
templates/
├── 情感类/
│   ├── 半路截胡.txt
│   ├── 雨中告别.txt
│   ├── 擦肩回眸.txt
│   ├── 咖啡馆送花.txt
│   ├── 深夜车站.txt
│   └── ...
├── 动作类/
│   ├── 高空跳跃.txt
│   ├── 慢镜头奔跑.txt
│   └── ...
├── 氛围类/
│   ├── 雨夜霓虹.txt
│   ├── 晨光厨房.txt
│   └── ...
└── 叙事类/
    ├── 回忆闪回.txt
    ├── 平行剪辑.txt
    └── ...
```

这些都是**你自己的视觉语言库**。
每做一个新项目，先翻模板库，能复用的直接用，
不能复用的再从头写 —— 写完也别浪费，`/shot save` 存进去。

---

## 核心价值

**不是帮你写出更好的 prompt，而是让你不再重复劳动。**

好的镜头看过就看过了，用过就用过了。
这套工具让它变成可复用的资产。

半年以后，你会感谢现在开始积累的自己。
