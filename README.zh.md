# Shot Template Studio

> AI 视频生成的可复用镜头模板库。
> 自然语言模板，通用于 Kling、Runway、Pika、Sora、Midjourney Video 等主流平台。

---

## 问题背景

AI 视频生成存在一个"记忆问题"。

每次构思一段复杂的多人物镜头——第一人称视角被人半路截胡、雨中无言告别、擦肩回眸的怔忡瞬间——都要从头写 prompt。你在一段 prompt 上投入的匠心，用完即焚。

**Shot Template Studio 把匠心沉淀为资产。**

一个镜头只需写一次。可以换角色、换场景、跨项目反复复用。

---

## 工作原理

镜头以带占位符的自然语言文本保存：

```
【半路截胡】

需要的图：
- {场景}：环境
- {角色B}：远处跑来的人
- {角色C}：截胡的人
- {角色D}：第一人称视角的"我"

---

以{场景}为背景。这是{角色D}的第一人称视角。

{角色B}从画面深处挥着手朝镜头小跑过来，景深聚焦在{角色B}身上。

{角色D}缓缓抬起左手，掌心朝上。

这时{角色C}的右手从画面左侧伸出来，握住{角色D}的左手。
紧接着{角色C}整个人从左侧走出来，身影遮挡住身后正在靠近的
{角色B}。景深拉近，聚焦到{角色C}。

{角色C}的上半身占据大半个屏幕，含情脉脉地看着镜头，
轻微歪头，淡淡一笑。然后双手捧起{角色D}的左手，
贴到自己的脸颊上，微微眯着眼睛，沉浸享受的样子。

---

适用：情感反转、意外介入、三角关系
时长参考：8-12秒
```

使用时提供四张参考图，skill 自动将占位符替换为图片描述，输出可直接使用的视频 prompt。

---

## 双向工作流

```
模板 → Prompt：
  /shot use <模板名>
    → 提供参考图
    → 得到可粘贴的 prompt

视频 → 模板：
  /shot extract <视频或描述>
    → AI 分析镜头结构
    → 生成可复用模板
    → 存入个人库
```

核心价值是这个闭环：**看到欣赏的镜头能积累下来，写过的好 prompt 能分享出去。**

## 命令

### `/shot list`
列出所有模板，显示标签与适用场景。

### `/shot use <模板名>`
交互式组装 prompt。依次为每个占位符提供参考图，skill 输出完整的自然语言 prompt，兼容主流视频生成模型。

### `/shot extract <视频源>`
分析视频文件或文字描述，产出新模板。AI 自动识别所需图像、镜头序列、节奏、适用场景标签。

### `/shot save <模板名>`
把当前对话中编写（或优化）过的 prompt 保存为可复用模板。自动把具体人物/场景抽象为占位符。

## 模板设计哲学

**自然语言，而非结构化数据。**

目标用户有两类：(a) 可能不写代码的创意从业者，(b) 训练数据分布是自然语言的视频生成模型。任何中间的结构化表示——JSON、YAML、DSL——都会给前者增加认知负担，给后者增加翻译开销。

**占位符使用母语。**

对中文编导而言，`{角色A}` 比 `{character_a}` 更直觉。模板天然支持多语言。

**编辑器友好。**

所有模板都是 `.txt` 文件。不熟悉 Git 的编导可以直接用记事本打开、调整措辞、保存使用。不会有解析错误，不会有 schema 校验失败。

**模型无关。**

编译出来的 prompt 是纯描述性文本，在 Kling、Runway Gen-3、Pika、Sora、海螺、Midjourney Video 上表现一致，无需针对特定模型微调。

## 模板市场愿景

单人维护的模板库是线性增长。共享市场是指数级增长。

想象你能浏览这样的精选合集：

- **王家卫风格** — 凝视特写、反射面、霓虹配色
- **诺兰风格** — IMAX 级实体构图、平行剪辑
- **韩剧合集** — 命运相遇、雨中告白、慢动作对视
- **纪录片合集** — 采访构图、B-roll 转场、切镜头

社区模板以 PR 形式提交到中央 `shot-template-marketplace` 仓库。策展人审核质量。用户浏览、安装。模板作者积累作品集，编导每个项目节省数小时。

共同原则：**把创作决策保存为可复用状态**。
