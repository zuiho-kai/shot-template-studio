# shot

镜头模板工作室。一个命令，两种方向。

## 命令总览

| 命令 | 方向 | 做什么 |
|------|------|--------|
| `/shot use [模板名]` | 模板 → prompt | 挑一个模板，填入图片，输出完整 prompt |
| `/shot list` | - | 列出所有可用模板 |
| `/shot extract [视频]` | 视频 → 模板 | 分析视频，拆解成一个可复用的模板 |
| `/shot save [模板名]` | - | 把当前对话里的 prompt 保存为新模板 |

---

## /shot list

列出 `templates/` 目录里所有的 `.txt` 文件。

显示格式：
```
可用镜头模板：
  1. 半路截胡    — 情感反转、意外介入
  2. 雨中告别    — 离别、遗憾、克制
  3. 擦肩回眸    — 命运感、初次相遇
  ...
```

每个模板读第一行【模板名】和"适用："那一行。

## /shot use [模板名]

用一个模板组装最终的视频 prompt。

### 流程

1. 读取 `templates/{模板名}.txt`
2. 提取文件里所有的占位符（{场景}、{角色A}、{角色B} 等）
3. 问用户：每个占位符对应哪张图？用户可以：
   - 拖入图片
   - 给图片路径
   - 给图片描述（比如"戴眼镜的短发女生"）
4. 把占位符替换成具体描述或图片标识
5. 输出最终的 prompt 文本

### 例子

```
用户: /shot use 半路截胡

你: 这个模板需要 4 张图：
    - {场景}: 场景图
    - {角色B}: 远处跑来的人
    - {角色C}: 从旁边截胡的人
    - {角色D}: 镜头视角的"我"
    
    请依次提供这 4 张图或描述。

用户: [提供 4 张图]

你: [输出替换好占位符的完整 prompt，可直接粘贴到 Kling/Runway/Pika]
```

### 输出格式

直接输出可用的自然语言 prompt，不要加结构化标记。
结尾附一句提示："可直接粘贴到文生视频模型。"

## /shot extract [视频路径或描述]

反向工作：看一段视频，拆解出其中的镜头语言，保存为可复用模板。

### 流程

1. 如果用户给的是视频文件路径：
   - 用多模态能力分析视频（抽关键帧、识别角色、识别动作）
2. 如果用户给的是自然语言描述：
   - 直接基于描述分析
3. 拆解出模板三要素：
   - **需要的图**（场景、几个角色）
   - **镜头序列**（自然语言描述，用 {占位符} 替代具体角色）
   - **适用场景 + 时长参考**
4. 生成符合 `templates/` 格式的 `.txt` 文件
5. 问用户起什么名字，保存到 `templates/{名字}.txt`

### 关键：保持自然语言

提取出来的模板不要写成分镜脚本格式。
就用流畅的中文描述，像模板库里现有的那样。

### 例子

```
用户: /shot extract 这段视频：一个女生在咖啡馆看书，
      男生推门进来环顾四周找到她，走过去把一束花放在桌上。

你: [分析后输出]

    【咖啡馆送花】

    需要的图：
    - 场景：咖啡馆内景
    - 角色A：看书的女生
    - 角色B：送花的男生

    ---

    以{场景}为背景。
    
    {角色A}坐在靠窗的位置，低头看书，手边放着一杯咖啡。
    画面安静，只有书页翻动的声音。
    
    镜头切到门口，{角色B}推门进来，带进一阵风，环顾四周。
    {角色B}的目光停在{角色A}身上，露出一点紧张的笑意。
    
    {角色B}穿过咖啡馆，走到{角色A}桌前。
    {角色A}察觉到有人，抬起头。
    
    {角色B}把一束花轻轻放在桌上，没有说话，只是微笑。
    镜头定格在{角色A}的表情上——从惊讶到温柔。
    
    ---
    
    适用：浪漫、惊喜、安静的情感瞬间
    时长参考：10-14秒

    要保存为模板"咖啡馆送花"吗？
```

## /shot save [模板名]

如果用户在对话里写了一段不错的 prompt，想保存下来，直接 save。

### 流程

1. 从当前对话里找到最近的一段视频 prompt
2. 让用户确认：这段是否要保存
3. 自动把具体的人物/场景替换成占位符
4. 问用户：
   - 模板名叫什么
   - 适用场景描述
5. 按格式保存到 `templates/{模板名}.txt`

---

## 模板文件格式

每个模板就是一个 `.txt` 文件，严格遵守这个结构：

```
【模板名】

需要的图：
- xxx
- xxx

---

[一段流畅的中文描述，用 {占位符} 表示角色和场景]

---

适用：[场景标签]
时长参考：[X-Y秒]
```

**原则**：
- 就是自然语言，不是 JSON
- 占位符用中文 `{角色A}`，不要用 `{char_a}`
- 不做"镜头语法"抽象，直接写"景深拉近、镜头切远景"这种编导看得懂的
- 编导能直接打开 `.txt` 手动改

## 工作流示例

### 正向：从模板做视频

```
/shot list
→ 看到"半路截胡"

/shot use 半路截胡
→ 按提示上传 4 张图
→ 得到完整 prompt
→ 粘贴到 Kling/Runway
→ 出视频
```

### 反向：从参考视频积累模板

```
看到一个抖音/B站上喜欢的镜头
/shot extract [视频]
→ 得到模板
/shot save 的自动保存
→ 以后自己用
```

### 长视频拼接

```
/shot use 擦肩回眸 → 得到 prompt A（前6秒）
/shot use 雨中告别 → 得到 prompt B（后12秒）
手动把 A 的结束画面作为 B 的起始画面
→ 拼成一个完整故事
```

---

## 设计原则

1. **编导友好**：模板就是 `.txt`，不是 JSON
2. **自然语言优先**：输入是中文，输出也是中文，符合模型训练分布
3. **双向可复用**：能从模板生成 prompt，也能从视频提取模板
4. **无侵入**：不依赖任何特定视频模型，输出是通用 prompt
5. **积累导向**：每次用过好的 prompt 都能变成资产
